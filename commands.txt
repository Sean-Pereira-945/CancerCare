### Step 2: Get auth token for experiment runner
Make sure backend is running, then either register or login.

Register (first time):
```powershell
$registerBody = @{
  name = "Research User"
  email = "research.user@example.com"
  password = "StrongPassword123!"
  cancer_type = "general"
  role = "patient"
} | ConvertTo-Json

$reg = Invoke-RestMethod -Method POST -Uri "http://127.0.0.1:8000/api/auth/register" -ContentType "application/json" -Body $registerBody
$TOKEN = $reg.access_token
$TOKEN
```

Login (if account already exists):
```powershell
$loginBody = @{
  email = "research.user@example.com"
  password = "StrongPassword123!"
} | ConvertTo-Json

$login = Invoke-RestMethod -Method POST -Uri "http://127.0.0.1:8000/api/auth/login" -ContentType "application/json" -Body $loginBody
$TOKEN = $login.access_token
$TOKEN
```

### Step 3: Run ablation experiments (writes CSV + raw JSON)

```

```

## 5) Tiny-subset research mode (Groq free-tier friendly)
This mode creates a small dataset from the first N lines of `eval_qa.jsonl` and runs experiments on it.

### Step A: Create tiny dataset (example: 20 items)
```powershell
cd C:\Users\SEAN\OneDrive\Desktop\CancerCare
Get-Content .\research\datasets\eval_qa.jsonl -TotalCount 20 | Set-Content .\research\datasets\eval_qa.tiny20.jsonl
```

For even smaller runs:
```powershell
# 10 items
Get-Content .\research\datasets\eval_qa.jsonl -TotalCount 10 | Set-Content .\research\datasets\eval_qa.tiny10.jsonl

# 5 items (smoke test)
Get-Content .\research\datasets\eval_qa.jsonl -TotalCount 5 | Set-Content .\research\datasets\eval_qa.tiny5.jsonl
```

### Step B: Copy matrix for tiny runs (to avoid overwriting main file)
```powershell
Copy-Item .\research\experiment_matrix.csv .\research\experiment_matrix.tiny.csv
```

### Step C: Run experiments on tiny dataset
```powershell
cd C:\Users\SEAN\OneDrive\Desktop\CancerCare\backend
.\venv\Scripts\Activate.ps1
python experiment_runner.py --token $TOKEN --api-base "http://127.0.0.1:8000" --dataset "..\research\datasets\eval_qa.tiny20.jsonl" --csv "..\research\experiment_matrix.tiny.csv"
```

### Step D: Preserve tiny raw output before next run
`experiment_runner.py` always writes to `research\experiment_raw_data.json`, so save a copy:
```powershell
Copy-Item ..\research\experiment_raw_data.json ..\research\experiment_raw_data.tiny20.json
```

### Step E: Statistical analysis + plots for tiny run
```powershell
cd ..\research
Copy-Item .\experiment_raw_data.tiny20.json .\experiment_raw_data.json -Force
python statistical_analysis.py
python generate_plots.py
```

Note:
- `generate_plots.py` reads `experiment_matrix.csv` by default. For tiny plotting, temporarily replace it:
```powershell
Copy-Item .\experiment_matrix.tiny.csv .\experiment_matrix.csv -Force
python generate_plots.py
```

## 6) Publishable-run protocol (new robust pipeline)
The runner now supports retries, resumable raw outputs, and controlled subsets.

### 6.1 Pilot run (paper dry-run, 50 items)
```powershell
cd C:\Users\SEAN\OneDrive\Desktop\CancerCare\backend
.\venv\Scripts\Activate.ps1
python experiment_runner.py `
  --token $TOKEN `
  --api-base "http://127.0.0.1:8000" `
  --dataset "..\research\datasets\eval_qa.jsonl" `
  --csv "..\research\experiment_matrix.pilot.csv" `
  --raw-out "..\research\experiment_raw_data.pilot.json" `
  --quality-report-out "..\research\experiment_quality_report.pilot.json" `
  --failure-log-out "..\research\experiment_failures.pilot.jsonl" `
  --judge-model "llama-3.3-70b-versatile" `
  --shuffle `
  --seed 42 `
  --max-items 50 `
  --request-retries 5 `
  --judge-retries 5 `
  --retry-delay-seconds 3 `
  --sleep-seconds 1.5 `
  --min-coverage-ok 0.90
```

### 6.2 Resume same run after quota reset
Use the same command again (same `--raw-out`), already processed items are skipped.

### 6.3 Full run (for paper tables)
```powershell
python experiment_runner.py `
  --token $TOKEN `
  --api-base "http://127.0.0.1:8000" `
  --dataset "..\research\datasets\eval_qa.jsonl" `
  --csv "..\research\experiment_matrix.paper.csv" `
  --raw-out "..\research\experiment_raw_data.paper.json" `
  --quality-report-out "..\research\experiment_quality_report.paper.json" `
  --failure-log-out "..\research\experiment_failures.paper.jsonl" `
  --judge-model "llama-3.3-70b-versatile" `
  --request-retries 5 `
  --judge-retries 5 `
  --retry-delay-seconds 3 `
  --sleep-seconds 1.5 `
  --min-coverage-ok 0.95
```

The run now also exports:
- Quality report JSON (coverage, status counts, failure reasons per variant)
- Failure log JSONL (one failed item per line for appendix/error analysis)
